## 🔄 版本发布流程

### 1. 开发阶段
- 在 `versions/X.X/source/` 中开发新功能
- 保持代码私有，不提交到公开仓库
- 进行内部测试和质量保证

### 2. 发布准备
- 编译生成发布包到 `versions/X.X/release/`
- 更新 `versions/X.X/changelog.md`
- 准备营销材料和文档

### 3. 商业发布
- 通过付费渠道分发 (QQ群、官网等)
- 激活许可证系统
- 提供技术支持

### 4. 开源转换
- 当版本变为免费时，将源代码移动到主目录
- 更新 MIT 许可证头部
- 提交到公开 Git 仓库
